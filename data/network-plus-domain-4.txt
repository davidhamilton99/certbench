B) Integrity
C) Availability

A) Zero trust
B) Least privilege
C) Defense in depth
D) Separation of duties
Correct: C

The concept is analogous to a medieval castle with a moat, walls, guards, and locked doors — each layer adds friction for an attacker.

Question: An attacker sends a massive volume of SYN packets to a web server without completing the TCP three-way handshake, exhausting the server's connection table. Which attack is this?
A) UDP flood
B) Ping of death
C) SYN flood
D) Amplification attack
Correct: C
Explanation: A SYN flood exploits the TCP handshake by sending SYN packets and never responding to the server's SYN-ACK, filling the half-open connection table until the server can no longer accept legitimate connections. A UDP flood sends large volumes of UDP datagrams but does not exploit the TCP handshake mechanism. SYN flood attacks are mitigated by SYN cookies, which allow the server to validate connection requests without storing state for each half-open connection.

Question: An attacker sends a crafted DNS query to a large open DNS resolver with the victim's IP as the source. The resolver responds with a DNS answer that is 50 times the size of the query. What attack technique is being exploited?
A) DNS zone transfer
B) DNS hijacking
C) DNS amplification
D) DNS poisoning
Correct: C
Explanation: DNS amplification exploits the size difference between a small DNS query and a large DNS response to multiply attack traffic, with the DNS resolver unwittingly sending large responses to the spoofed victim IP. DNS poisoning corrupts the resolver's cache with false records rather than using it as a reflection source. DNS amplification is mitigated by disabling open DNS resolvers, implementing Response Rate Limiting (RRL), and using BCP38 to prevent IP spoofing on ISP networks.

A) SYN flood
B) DNS poisoning
C) Distributed Denial of Service (DDoS)

Correct: C

A) Port security
B) Native VLAN reassignment
C) DHCP snooping
D) Dynamic ARP inspection
Correct: B
Explanation: Reassigning the native VLAN away from VLAN 1 reduces the risk of VLAN hopping attacks that rely on double-tagging using the default native VLAN, because frames tagged with a non-default VLAN are less likely to be processed incorrectly.
