Question: A technician is troubleshooting a VM that fails to start and receives an error stating the host system does not support hardware virtualization. Where should the technician go to resolve this issue?
A) Device Manager in the host OS
B) The hypervisor's virtual NIC settings
C) BIOS/UEFI firmware settings to enable VT-x or AMD-V
D) The guest VM's display adapter settings
Correct: C
Explanation: Hardware virtualization extensions (Intel VT-x or AMD-V) are enabled or disabled in the system's BIOS/UEFI firmware settings. If the extensions are present on the CPU but disabled in firmware, the hypervisor will report that hardware virtualization is not supported and 64-bit VMs will fail to start. The technician should reboot the host, enter BIOS/UEFI setup, and enable the virtualization setting.

B) Software that creates and manages virtual machines by abstracting physical hardware resources
C) A network protocol used to connect VMs to the internet

Correct: B

Hypervisors are the foundational technology enabling all forms of server and desktop virtualization.

Question: A junior technician asks why their VirtualBox VM for testing runs noticeably slower than the host OS. Which is the MOST likely explanation?
A) VirtualBox is a Type 1 hypervisor with high overhead
B) The VM shares host CPU, RAM, and disk I/O with the host OS and other processes, creating resource contention
C) VMs always run at half the speed of the host hardware by design
D) The virtual NIC limits the VM's processing speed

Allocating more RAM, closing unnecessary host applications, and using an SSD can help improve VM performance.

B) Assign only as many vCPUs as the workload requires to avoid unnecessary CPU contention across VMs
