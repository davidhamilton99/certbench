Question: A company donates old laptops to a charity. The laptops previously held confidential internal documents. Which data sanitization step is REQUIRED before donation?
A) Deleting documents and donating the laptops as-is
B) Performing a cryptographic erase or multi-pass overwrite on all drives so no company data is recoverable
C) Installing a fresh operating system without wiping the original data partition
D) Encrypting the data before donation so the charity cannot read it
Correct: B
Explanation: Donating assets containing company data without sanitization creates a data breach risk. Cryptographic erase or multi-pass overwrite ensures data is unrecoverable. Simply deleting files or installing a new OS leaves original data recoverable with forensic tools. Encrypting data leaves it intact and potentially recoverable if the key is exposed.

Domain: 4.0 Security Operations
Sub-objective: 4.3 — Explain various activities associated with vulnerability management
Question: An organisation's vulnerability management programme identifies a critical vulnerability in a web application framework used across 47 applications. Patching all 47 simultaneously would take six weeks. Which remediation strategy should be applied FIRST to the highest-risk applications?
A) Wait until all 47 applications can be patched simultaneously
B) Apply a virtual patch (WAF rule) to block exploitation while prioritising and scheduling application patches by risk
C) Disable all 47 applications until patching is complete
D) Accept the risk for all 47 applications and document exceptions
Correct: B
Explanation: A virtual patch (WAF rule blocking the exploitation path) is a compensating control that reduces immediate risk while the team works through the patching backlog ordered by risk. Waiting six weeks to patch exposes all applications. Disabling all applications causes unnecessary outage. Accepting risk without any compensating control for critical vulnerabilities is imprudent.

Domain: 4.0 Security Operations
Sub-objective: 4.4 — Explain security alerting and monitoring concepts and tools
Question: A SIEM rule fires when more than five distinct privileged commands are executed within 60 seconds by a non-administrative account. This triggers an alert but also automatically creates an incident ticket and pages the on-call engineer. Which monitoring and response approach is described?
A) Passive monitoring with manual ticketing
B) Alert-driven automation integrating SIEM with SOAR for escalation
C) Real-time firewall rule modification
D) Threat intelligence enrichment
Correct: B
Explanation: Automatically creating a ticket and paging the on-call engineer in response to a SIEM alert is SOAR orchestration integrated with monitoring — removing manual steps from the escalation path. This is not passive monitoring (it takes automated action). Firewall rule modification is a separate response action. Threat intelligence enrichment adds context; it does not escalate.

Domain: 4.0 Security Operations
Sub-objective: 4.5 — Given a scenario, modify enterprise capabilities to enhance security
Question: An enterprise is deploying a zero trust architecture and wants to ensure that every application access request is evaluated based on user identity, device health, and application sensitivity — regardless of whether the request originates from inside or outside the corporate network. Which capability is CENTRAL to this architecture?
A) Traditional perimeter firewall with DMZ
B) Continuous contextual access evaluation using an identity-aware proxy and policy engine
C) VPN with split tunnelling disabled
D) Static IP allowlisting for all application servers
Correct: B
Explanation: Zero trust requires continuous verification of identity, device posture, and context for every access request. An identity-aware proxy and policy engine implements per-request access decisions rather than implicit network-based trust. Traditional perimeter firewalls grant broad network trust after authentication. VPN improves encryption but is not per-request contextual. Static IP allowlisting is network-based and ignores identity and device health.

Domain: 4.0 Security Operations
Sub-objective: 4.6 — Given a scenario, implement and maintain identity and access management
Question: A company uses OpenID Connect (OIDC) to allow users to log in to its application using their Google account. Which IAM function does OIDC provide in this scenario?
A) Delegated authorisation for third-party API access
B) Federated authentication — verifying user identity via a trusted external identity provider
C) Mandatory access control policy enforcement
D) Privileged session management
Correct: B
Explanation: OIDC is an identity layer built on OAuth 2.0 that provides authentication — verifying who the user is via a trusted IdP (Google). OAuth 2.0 itself provides delegated authorisation (what resources the app can access). MAC is a government-grade access model. PSM records privileged sessions.

Domain: 4.0 Security Operations
Sub-objective: 4.7 — Explain the importance of automation and orchestration related to secure operations
Question: A security team is evaluating whether to automate firewall block actions in response to threat intelligence feeds. A key concern is accidentally blocking legitimate business traffic. Which control BEST mitigates this risk while preserving the benefits of automation?
A) Disable the automation entirely and rely on manual blocking
B) Implement a human-in-the-loop approval step for high-impact block actions, with full automation for lower-risk actions
C) Block all threat intelligence feed IPs without review to maximise speed
D) Use the threat intelligence feed only for reporting, never for action
Correct: B
Explanation: A tiered approach — full automation for lower-risk actions (e.g., alerting) and a human-in-the-loop for high-impact actions (blocking) — preserves automation speed benefits while adding accountability for decisions with significant blast radius. Disabling automation sacrifices speed. Blocking all IPs without review risks false positives. Using feeds only for reporting wastes their actionable potential.

Domain: 4.0 Security Operations
Sub-objective: 4.8 — Explain appropriate incident response activities
Question: A forensic investigator is asked to collect evidence from a live Windows server that is suspected of being compromised. The investigator must collect a memory dump, running process list, and open network connections BEFORE powering off the server. Which forensic principle drives this decision?
A) Chain of custody documentation
B) Order of volatility — collecting the most transient evidence before it is lost
C) Legal hold compliance
D) E-discovery production requirements
Correct: B
Explanation: The order of volatility mandates collecting the most short-lived data first (RAM, process table, network connections) because this data is lost on shutdown. Chain of custody documents the evidence handling process. Legal hold preserves data. E-discovery produces evidence for legal proceedings. These all occur after collection.

Domain: 4.0 Security Operations
Sub-objective: 4.9 — Given a scenario, use data sources to support an investigation
Question: An analyst is correlating logs to investigate a lateral movement incident. Which combination of log sources BEST reveals an attacker moving from a compromised workstation to additional internal servers using stolen credentials?
A) Web proxy logs and vulnerability scan reports
B) Windows authentication event logs (4624/4648), firewall logs for internal east-west traffic, and Sysmon process creation logs
C) SNMP trap logs and NetFlow external traffic
D) Application error logs and DNS logs
Correct: B
Explanation: Lateral movement via stolen credentials is detected by correlating Windows logon events (4624 remote logons, 4648 explicit credential use), east-west firewall logs showing internal connections, and process creation logs showing tools executed on target systems. Web proxy and vulnerability scans focus on internet traffic and weakness identification. SNMP and NetFlow external traffic miss internal lateral movement. Application errors and DNS logs provide limited lateral movement visibility.

Domain: 4.0 Security Operations
Sub-objective: 4.3 — Explain various activities associated with vulnerability management
Question: A security analyst discovers a CVSS 4.3 (Medium) vulnerability on a web server. However, a public exploit is actively being used in the wild against this exact vulnerability. Which factor MOST changes the effective priority of this finding?
A) The CVSS base score should be raised by the analyst
B) The presence of active exploitation in the wild elevates the effective risk, requiring accelerated remediation despite the medium base score
C) Medium-severity vulnerabilities should always wait for the next quarterly patching cycle
D) The vulnerability can be ignored if the web server is behind a load balancer
Correct: B
Explanation: CVSS base scores do not account for real-world exploitation activity. Active exploitation (captured in CVSS temporal metrics as Exploit Code Maturity: High) dramatically increases the likelihood of compromise, overriding the relatively low base score. Quarterly patching for an actively exploited vulnerability is unacceptable. A load balancer does not remove the vulnerability.
