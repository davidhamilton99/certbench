Sub-objective: 5.3 — Troubleshoot common wireless connectivity issues
Question: A user's laptop connects to the Wi-Fi access point at the far end of the building rather than the closer one, resulting in poor performance. The user physically walks past the closer AP without roaming. What wireless behavior causes this issue?
A) Band steering pushing the client to 2.4 GHz
B) Client stickiness — the device holds on to the original AP association even when a better AP is available
C) Co-channel interference from the far AP reducing the near AP's effectiveness
D) The near AP has reached maximum client capacity
Correct: B
Explanation: Client stickiness occurs because the roaming decision is made by the client device, not the AP — as long as the original AP's signal is above the minimum threshold (even if poor), some clients will not roam to a stronger AP. Band steering (option A) affects which frequency band is used, not which AP is associated. Co-channel interference (option C) is a separate channel planning issue. Client stickiness is addressed with 802.11v BSS Transition Management, which allows APs to suggest or force clients to roam to better APs.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.2 — Troubleshoot common cable connectivity issues
Question: A newly installed Cat6A cable run fails the alien crosstalk (AXT) test during certification. What does alien crosstalk refer to?
A) Crosstalk between pairs within the same cable
B) Interference between adjacent cables in the same conduit or bundle
C) Signal reflection at the far end of the cable
D) Noise induced by external EMI sources such as fluorescent lights
Correct: B
Explanation: Alien crosstalk (AXT) is interference that couples between separate cables running in close proximity — unlike NEXT/FEXT which occur within the same cable's pairs, AXT occurs between different cables in the same conduit or cable tray. Signal reflections (option C) are measured as return loss. External EMI (option D) is interference from non-cable sources. Cat6A cabling standard addresses AXT through improved shielding and is particularly important in dense structured cabling installations where many cables run in parallel.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.5 — Troubleshoot common network service issues
Question: A network administrator receives alerts that devices on the network are sending DNS queries to an unauthorized external DNS server (8.8.4.4) instead of the internal DNS server. This bypasses internal DNS filtering policies. What is the most effective network-level control to enforce use of the internal DNS server?
A) Send an email to all users reminding them to use the correct DNS server
B) Configure a firewall rule blocking outbound UDP/TCP port 53 to all destinations except the internal DNS server
C) Increase the DHCP lease time to prevent DNS server changes
D) Deploy an additional DHCP server to ensure all clients get correct DNS settings
Correct: B
Explanation: Blocking outbound port 53 (DNS) to all destinations except the authorized internal DNS server at the firewall forces all DNS traffic through the controlled path, preventing bypassing of DNS security controls regardless of client configuration. Emails to users (option A) cannot enforce technical compliance. DHCP lease time (option C) is unrelated to DNS enforcement. An additional DHCP server (option D) ensures clients get correct settings initially but does not prevent manual DNS overrides. Firewall-enforced DNS redirection is a standard enterprise security control.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.1 — Explain the network troubleshooting methodology
Question: Which statement BEST describes the purpose of the "establish a theory of probable cause" step in the CompTIA troubleshooting methodology?
A) To immediately implement the most likely solution
B) To form a testable hypothesis about the root cause based on gathered information
C) To document the resolution steps for future reference
D) To verify that the implemented solution has resolved the problem
Correct: B
Explanation: Establishing a theory means forming a logical, testable hypothesis about the most probable root cause before taking any action — this ensures that subsequent tests are focused and relevant rather than random. Implementing solutions (option A) comes after the theory is tested and confirmed. Documentation (option C) is the final step. Verification (option D) follows implementation. A well-formed theory guides efficient troubleshooting and prevents "swapping parts randomly" which wastes time and can introduce new problems.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.4 — Troubleshoot general networking issues
Question: A network engineer adds a new static route to a router. Shortly after, users on multiple subnets begin experiencing connectivity issues. The engineer suspects the new route is causing problems. What troubleshooting step should be taken immediately?
A) Reboot all affected routers
B) Remove or roll back the new static route and verify connectivity is restored
C) Add additional static routes to compensate
D) Increase the router's routing table size
Correct: B
Explanation: When a configuration change is the suspected cause of a problem, rolling back the change is the fastest way to restore service and confirm causation — this is part of establishing and testing a theory. Adding more routes (option C) could worsen an already confused routing table. Rebooting routers (option A) is a drastic step that should not be done until simpler reversals are tried. The correlation between the new route and the problem start time is strong evidence that the route introduction was the cause.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.5 — Troubleshoot common network service issues
Question: A user running a VPN client can connect to the VPN but cannot reach internal network resources after connecting. Other VPN users have no issues. The user's VPN client shows a valid internal IP address assigned. What is the most likely cause?
A) The VPN server certificate has expired
B) Split tunneling is configured incorrectly on this user's VPN profile
C) The user's account does not have the correct access permissions or firewall rules in the VPN policy
D) The VPN server's DHCP scope is exhausted
Correct: C
Explanation: If the user receives a valid IP from the VPN but cannot reach internal resources while others can, the issue is likely authorization rather than connectivity — the user's VPN group policy, firewall ACL, or network access control may not permit access to the needed resources. Split tunneling misconfiguration (option B) would cause all traffic to route through the VPN rather than selected traffic, which would make external browsing fail rather than internal resources. An expired certificate (option A) would prevent VPN connection establishment. VPN access troubleshooting must consider both connectivity (can connect) and authorization (can access resources).

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.4 — Troubleshoot general networking issues
Question: A network switch port is configured for voice VLAN 10 and data VLAN 20. A VoIP phone connected to that port receives an IP address but the voice quality is extremely poor with high jitter. Standard VLAN configuration is verified as correct. What should be checked next?
A) Whether QoS (Quality of Service) policies are configured to prioritize voice traffic on the VLAN
B) Whether the DNS server for VLAN 10 is configured correctly
C) Whether the switch port is operating at 100 Mbps instead of 1 Gbps
D) Whether the NTP server is synchronized for timestamp accuracy
Correct: A
Explanation: VoIP traffic is highly sensitive to jitter and delay — without QoS policies that prioritize voice traffic (typically using DSCP EF / DSCP 46 and priority queuing), voice packets compete equally with data packets and experience high jitter during congestion. DNS configuration (option B) affects name resolution, not call quality. Port speed (option C) at 100 Mbps is more than sufficient for VoIP. NTP (option D) affects timestamps in logs, not real-time voice packet delivery. QoS is the foundational requirement for enterprise VoIP deployments.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.3 — Troubleshoot common wireless connectivity issues
Question: A wireless client fails to associate with an AP. The client's wireless logs show the error "802.11 authentication failed — station not permitted." The user's credentials are correct and have been verified. What is the most likely cause?
A) The AP is using WEP instead of WPA2
B) The client's MAC address has been added to the AP's MAC address deny list
C) The AP channel is set to an incorrect frequency for the client's region
D) The DHCP scope is exhausted, preventing the client from getting an IP
Correct: B
Explanation: "Station not permitted" errors at the 802.11 authentication stage (before 802.1X or passphrase verification) typically indicate that MAC filtering is blocking the client — the client's MAC address is either on a deny list or is missing from a required allow list. WEP vs. WPA2 (option A) would cause an authentication protocol mismatch error. DHCP exhaustion (option D) occurs after successful wireless association, not before. MAC address filtering is a legacy security measure that is increasingly replaced by 802.1X port-based authentication.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.2 — Troubleshoot common cable connectivity issues
Question: A technician uses a multimeter to test a patch cable and measures 0 ohms between the two ends of a conductor pair. What does this measurement indicate?
A) The conductor pair has excessive resistance and should be replaced
B) The conductor pair has continuity — a good, unbroken connection
C) The conductor pair is shorted to an adjacent pair
D) The cable is properly terminated with the correct impedance
Correct: B
Explanation: 0 ohms resistance between both ends of a conductor means there is a complete, unbroken metallic path — continuity is confirmed. Excessive resistance (option A) would show a measurable non-zero value, not zero. A short to an adjacent pair (option C) would show continuity between conductors that should not be connected. While a multimeter confirms continuity, it cannot detect wiring order faults or measure high-frequency performance — cable testers are still needed for complete cable verification.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.5 — Troubleshoot common network service issues
Question: After a power outage, a network engineer finds that multiple servers cannot communicate with the Active Directory domain. The engineer suspects the issue is related to the time offset caused by the power outage. What should the engineer do to restore authentication?
A) Reinstall the network drivers on all affected servers
B) Synchronize the time on all affected servers with an authoritative NTP source or the domain PDC Emulator
C) Regenerate the SSL certificates on all affected servers
D) Flush the DNS cache on all affected servers
Correct: B
Explanation: After a power outage, servers' hardware clocks may drift or reset — Kerberos authentication requires clocks to be within 5 minutes of the domain controller, so resynchronizing all server clocks with the domain's authoritative NTP source (PDC Emulator) will restore Active Directory authentication. Reinstalling drivers (option A) would not address a time synchronization issue. Certificate regeneration (option C) is unrelated to clock drift. DNS flushing (option D) is a good practice but would not resolve time-based Kerberos failures. The PDC Emulator role holder is the authoritative NTP source for the entire Active Directory domain.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.4 — Troubleshoot general networking issues
Question: A technician examines interface counters on a switch and finds a port accumulating "runts" — frames smaller than 64 bytes. What is the most common cause of runts on an Ethernet interface?
A) MTU set too high on the connected device
B) Duplex mismatch causing late collisions that truncate frames
C) Excessive cable length causing signal attenuation
D) VLAN misconfiguration adding extra bytes to frames
Correct: B
Explanation: Runts are undersized frames often resulting from collisions — in a duplex mismatch scenario, the half-duplex side detects collisions and sends a jam signal that truncates frames, creating runts. MTU set too high (option A) would cause giant frames (jumbo frames), not runts. Cable attenuation (option C) causes CRC errors, not specifically runts. VLAN tagging (option D) adds bytes (making frames larger), not smaller. The combination of runts, late collisions, and declining throughput is the classic symptom profile of a duplex mismatch.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.3 — Troubleshoot common wireless connectivity issues
Question: A wireless network is deployed in a large office with multiple APs all configured to the same SSID. Users on Windows laptops complain that their devices take a very long time to connect to the network in the morning. Investigation reveals the devices are attempting to connect to APs that are far away with weak signal before eventually settling on a closer AP. What feature, if enabled, would help this issue?
A) Band steering
B) 802.11r Fast BSS Transition
C) 802.11k Neighbor Reports allowing clients to discover nearby AP options faster
D) WPA3 enterprise authentication
Correct: C
Explanation: 802.11k provides neighbor report elements that give clients a list of nearby APs with signal information, allowing clients to make faster and better-informed roaming decisions rather than scanning all channels to find available APs. 802.11r (option B) speeds up re-authentication during roaming but does not help with initial AP selection. Band steering (option A) moves clients between frequency bands. WPA3 (option D) is a security standard. 802.11k reduces initial scan time and roaming delays by giving the client a pre-built neighbor list.

Domain: 5.0 Network Troubleshooting
Sub-objective: 5.5 — Troubleshoot common network service issues
Question: A network engineer troubleshooting FTP connectivity discovers that passive mode FTP works but active mode FTP fails. What is the likely cause?
A) The FTP server's certificate has expired
B) In active mode, the server initiates a connection back to the client, which is blocked by the client-side firewall or NAT
C) The FTP server does not support active mode
D) Active mode requires UDP while passive mode uses TCP
Correct: B
Explanation: In active FTP, the client specifies a port and the server initiates a data connection back to the client — firewalls and NAT devices that block unsolicited inbound connections will block this server-initiated connection. Passive FTP has the client initiate both the control and data connections, making it NAT and firewall friendly. Both active and passive FTP use TCP (option D is incorrect). Active mode FTP failure behind NAT or stateful firewalls is a classic networking problem, which is why passive mode has become the default for most FTP clients.
